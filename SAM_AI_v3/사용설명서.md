# SAM AI v3 사용설명서

## 시작하기

### 접속 방법
브라우저에서 다음 주소로 접속합니다:
```
http://localhost:5050
```

---

## 화면 구성

### 1. 사이드바 (왼쪽)
- **New Chat**: 새 대화 시작
- **대화 목록**: 이전 대화 기록
- **Settings**: API 키 및 환경설정

### 2. 채팅 영역 (가운데)
- 메시지 입력창
- AI 응답 표시 영역
- 파일 첨부 버튼

### 3. 상단 바
- AI 프로바이더 선택
- 채팅 모드 선택

---

## API 키 설정

처음 사용 시 Settings에서 API 키를 설정해야 합니다.

### 설정 방법
1. 왼쪽 하단 **Settings** 클릭
2. 각 프로바이더의 API 키 입력:
   - **Claude**: Anthropic API 키 (`sk-ant-...`)
   - **OpenAI**: OpenAI API 키 (`sk-...`)
   - **Gemini**: Google AI API 키 (`AI...`)
   - **Ollama**: 로컬 실행이므로 키 불필요

3. **Save** 버튼 클릭

### API 키 발급처
| 프로바이더 | 발급 사이트 |
|-----------|------------|
| Claude | https://console.anthropic.com |
| OpenAI | https://platform.openai.com/api-keys |
| Gemini | https://aistudio.google.com/apikey |

## 채팅 모드

### 1. Single 모드
- **용도**: 하나의 AI와 대화
- **사용법**:
  1. 상단에서 원하는 AI 선택 (Claude, GPT, Gemini, Ollama)
  2. 메시지 입력 후 전송
- **특징**: 가장 기본적인 대화 모드

### 2. Compare 모드
- **용도**: 여러 AI의 응답을 비교
- **사용법**:
  1. 상단에서 "Compare" 모드 선택
  2. 메시지 입력 후 전송
  3. 모든 온라인 AI가 동시에 응답
- **특징**:
  - AI별 응답이 나란히 표시됨
  - 같은 질문에 대한 다양한 관점 확인 가능

### 3. Smart 모드
- **용도**: 최적의 통합 답변 생성
- **사용법**:
  1. 상단에서 "Smart" 모드 선택
  2. 메시지 입력 후 전송
- **동작 원리**:
  1. **Phase 1**: 모든 AI에게 질문 전송
  2. **Phase 2**: 응답 수집 후 하나의 최적 답변으로 통합
- **특징**: 여러 AI의 장점을 결합한 답변

## 파일 첨부

### 지원 파일 형식

| 유형 | 확장자 |
|------|--------|
| 이미지 | .png, .jpg, .jpeg, .gif, .webp |
| 문서 | .pdf, .txt, .md, .csv, .docx |

### 첨부 방법
1. 메시지 입력창 옆 **파일 아이콘** 클릭
2. 파일 선택
3. 메시지와 함께 전송

### 제한 사항
- 최대 파일 크기: **20MB**
- 이미지는 Vision 지원 AI에서만 분석 가능

## 대화 관리

### 새 대화 시작
- 사이드바 상단의 **New Chat** 클릭

### 대화 기록 보기
- 사이드바의 대화 목록에서 원하는 대화 클릭

### 대화 삭제
- 대화 항목 위에 마우스를 올리면 나타나는 **삭제 아이콘** 클릭

### 대화 제목 변경
- 대화 항목 위에서 **편집 아이콘** 클릭
- 새 제목 입력 후 저장

## 응답 제어

### 응답 중단
- 응답 생성 중 **Stop** 버튼 클릭

### 코드 복사
- 코드 블록 우측 상단의 **Copy** 버튼 클릭

## Ollama (로컬 AI) 사용

Ollama는 인터넷 없이 로컬에서 실행되는 무료 AI입니다.

### 사전 요구사항
1. Ollama 설치: `brew install ollama`
2. 서버 시작: `brew services start ollama`
3. 모델 다운로드: `ollama pull gemma3:4b`

### 장점
- 무료 (API 비용 없음)
- 오프라인 사용 가능
- 데이터 프라이버시

### 단점
- 로컬 컴퓨터 성능에 의존
- 대형 모델은 느릴 수 있음

## 문제 해결

### AI가 응답하지 않음
1. Settings에서 API 키 확인
2. 프로바이더 상태 확인 (온라인/오프라인)
3. 인터넷 연결 확인

### Ollama가 오프라인으로 표시됨
```bash
# Ollama 서비스 상태 확인
brew services list | grep ollama

# 서비스 재시작
brew services restart ollama
```

### 파일 업로드 실패
- 파일 크기 20MB 이하인지 확인
- 지원 형식인지 확인

### 응답이 느림
- Compare/Smart 모드는 여러 AI 호출로 시간이 더 걸림
- Ollama는 로컬 하드웨어 성능에 따라 다름
