## iPad 테스트 + 배포 이슈 해결 — 세션 5

### 문제 1: 포트 5060 방화벽
- Windows 방화벽에 "SAM AI v3" TCP 5060 인바운드 규칙 추가
- `Python 3.11 Allow` 프로그램 기반 규칙 추가
- **결과**: iPad에서 포트 5060 접속 불가 (원인 불명, 방화벽 규칙은 정상)

### 해결: 포트 5050으로 변경
- `config.py` PORT=5060 → 5050 변경
- v2 서버 중지 후 v3를 5050에서 실행
- iPad에서 `http://192.168.0.83:5050` 접속 성공

### 문제 2: v2 서버가 백그라운드에서 동시 실행
- v2 (`SAM_AI_v2/app.py`)가 포트 5050에서 여전히 실행 중이었음
- v3도 5050에서 시작했지만 iPad는 먼저 바인딩된 v2 서버에 연결
- **해결**: v2 프로세스 완전 종료 후 v3만 실행

### 문제 3: Service Worker 캐시 (v2 → v3 전환)
- iPad Safari가 이전 v2 Service Worker 캐시에서 파일 제공
- v3 코드로 서버가 바뀌었지만 브라우저는 캐시된 v2 HTML/JS 표시

#### 적용한 수정:
1. `index.html` — 모든 CSS/JS 참조에 `?v=3` 캐시 버스팅 쿼리 추가
2. `app.py` — `GET /` 라우트에 `Cache-Control: no-cache, no-store` 헤더 추가
3. `sw.js` — cache-first → **network-first** 전략 변경, 모든 이전 캐시 삭제
4. `app.py` — `GET /api/clear-cache` 캐시 삭제 전용 페이지 추가
   - `/api/` 경로는 이전 SW가 가로채지 않으므로 확실히 서버에서 응답
   - Service Worker 해제 + 캐시 삭제 → 자동 리다이렉트

### 문제 4: Smart Mode synthesizer 에러 처리
- Claude API 크레딧 부족 시 에러가 텍스트 토큰으로 반환됨 (`[Claude Error: ...]`)
- 에러 응답이 수집 결과에 포함되어 synthesizer에도 전달

#### 적용한 수정 (`app.py` Smart 엔드포인트):
1. `errored` 집합으로 에러 프로바이더 추적
2. 수집 후 응답 내용에 `[Error:` 포함 여부 검사 → 자동 제외
3. Synthesizer 선택 시 에러 프로바이더 `exclude` 파라미터로 제외
4. 성공한 프로바이더가 1개뿐이면 종합 없이 해당 응답 직접 사용
