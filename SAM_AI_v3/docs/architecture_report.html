2. 단계별 처리 과정

사용자가 "안녕하세요"를 입력하고 Claude를 선택했을 때, 내부에서 일어나는 일.

1
사용자 입력
사용자가 메시지를 입력하고 전송 버튼을 누릅니다.
프론트엔드가 선택된 프로바이더(Claude)와 모델(claude-sonnet-4)을 확인합니다.
chat.js → sendMessage()
2
대화 생성 & SSE 요청
대화가 없으면 새 대화를 생성합니다 (POST /api/conversations).
XHR로 POST /api/chat에 SSE 스트리밍 요청을 보냅니다.
전송 데이터: { conversation_id, message, provider: "claude", model }
api.js → chatStream() api.js → createConversation()
3
서버: 메시지 저장 & 스레드 생성
Flask 서버가 요청을 수신합니다.
① 사용자 메시지를 대화 파일에 저장 (JSON)
② 전체 대화 이력을 AI 메시지 형식으로 변환
③ 별도 스레드에서 AI 호출 시작 (메인 스레드는 SSE 스트리밍 담당)
app.py → chat() conversation_manager.py → add_message()
4
AI Dispatcher → Claude 서비스
AI Dispatcher가 "claude"를 확인하고 ClaudeService로 라우팅합니다.
① API 키 확인 (config.json에서 로드)
② 메시지를 Anthropic 형식으로 변환 (system은 별도 파라미터)
③ anthropic SDK의 stream API 호출
④ 토큰 단위로 yield (제너레이터)
ai_dispatcher.py → stream_chat() claude_service.py → stream_chat()
5
실시간 스트리밍 전송
AI가 생성한 토큰이 Queue를 통해 SSE 스트림으로 전달됩니다.
각 토큰: data: {"token": "안녕"} 형식으로 전송.
프론트엔드가 실시간으로 화면에 표시하며 커서가 깜빡입니다.
app.py → sse_stream() chat.js → appendToken()
6
완료 & 저장
AI 응답이 끝나면:
① 전체 응답 텍스트를 대화 파일에 저장
② 첫 번째 대화면 자동으로 제목 생성
③ data: {"done": true} 전송
④ 프론트엔드에서 마크다운 렌더링 + 코드블록 하이라이트 적용
app.py → _save_response() chat.js → finalizeStream() markdown.js → render()

6. SSE 스트리밍 원리

실시간 응답이 어떻게 한 글자씩 화면에 나타나는지 설명합니다.

⚙

AI 스레드

별도 스레드에서 AI API를 호출합니다.

토큰이 생성될 때마다 Queue에 넣습니다.

q.put(('token', '안녕'))
📦

Queue (중계)

스레드 간 안전한 데이터 전달 통로.

AI 스레드 → Queue → SSE 스트림

('token', '안녕')
('token', '하세요')
('done', None)
📡

SSE 스트림

Queue에서 꺼내 SSE 형식으로 전송합니다.

브라우저가 XHR onprogress로 수신.

data: {"token":"안녕"}
data: {"token":"하세요"}
data: {"done":true}
왜 이렇게 복잡한가? iPad Safari는 EventSource(GET만 지원)를 사용할 수 없어서, XHR POST + SSE 방식을 사용합니다.
