시작하기

SAM AI v3는 여러 AI 서비스를 하나의 인터페이스에서 사용할 수 있는 멀티 AI 채팅 애플리케이션입니다.

지원 AI 프로바이더

프로바이더 모델 특징
Claude claude-sonnet-4, claude-haiku-4 Anthropic AI, 뛰어난 분석력
GPT gpt-4o, gpt-4o-mini, gpt-4.1 OpenAI, 범용적 성능
Gemini gemini-2.5-flash/pro, gemini-2.0 Google AI, 빠른 응답
Ollama gemma3:4b 등 로컬 AI, 무료, 오프라인

API 키 설정

처음 사용 시 Settings에서 API 키를 설정해야 합니다.

설정 방법
  1. 왼쪽 하단 Settings 버튼 클릭
  2. 각 프로바이더의 API 키 입력:
    • Claude: sk-ant-... 형식
    • OpenAI: sk-... 형식
    • Gemini: AI... 형식
    • Ollama: API 키 불필요 (로컬)
  3. Save API Keys 버튼 클릭

API 키 발급처

프로바이더 발급 사이트
Claude https://console.anthropic.com
OpenAI https://platform.openai.com/api-keys
Gemini https://aistudio.google.com/apikey

채팅 모드

Single 모드

선택한 하나의 AI와 대화합니다. 가장 기본적인 모드입니다.

  • 상단에서 원하는 AI 프로바이더 선택
  • 모델도 선택 가능 (기본값 자동 적용)
  • 메시지 입력 후 전송
Compare 모드

모든 온라인 AI에게 동시에 질문하고 응답을 비교합니다.

  • 같은 질문에 대한 다양한 관점 확인
  • AI별 응답이 나란히 표시됨
  • 각 AI의 강점 비교 가능
Smart 모드

여러 AI의 응답을 수집하여 최적의 답변으로 통합합니다.

  • Phase 1: 모든 온라인 AI에게 질문 전송
  • Phase 2: 응답 수집 후 통합 답변 생성
  • Synthesizer 우선순위: Claude > Gemini > OpenAI

  • 제목 변경: 대화 항목에 마우스 오버 후 편집 아이콘

Ollama (로컬 AI) 상세 설명

Ollama는 로컬 컴퓨터에서 AI 모델을 실행하는 오픈소스 프로젝트입니다. 인터넷 연결 없이 완전히 오프라인으로 작동하며, API 비용이 발생하지 않습니다.

Ollama의 장점

  • 무료: API 호출 비용 없음
  • 오프라인 사용: 인터넷 연결 불필요
  • 데이터 프라이버시: 모든 데이터가 로컬에서 처리됨
  • 다양한 모델: Llama, Gemma, Mistral 등 다양한 오픈소스 모델 지원

시스템 요구사항

Ollama를 사용하려면 충분한 하드웨어 리소스가 필요합니다.

모델 필요 RAM 권장 사양
gemma3:4b (4B 파라미터) 약 3~4 GB 8GB RAM 이상
llama3:8b (8B 파라미터) 약 5~6 GB 16GB RAM 이상
llama3:70b (70B 파라미터) 약 40 GB 64GB RAM 이상

클라우드 서버에서 Ollama를 사용할 수 없는 이유

메모리 제한 문제

이 서버(do-server)의 사양:

  • 총 RAM: 3.8 GB
  • 사용 가능 RAM: 약 2.5 GB
  • CPU: 2코어

gemma3:4b 모델은 약 3~4 GB의 RAM이 필요하지만, 서버의 사용 가능한 메모리는 약 2.5 GB입니다. 또한 다른 웹 서비스들(약 15개)이 이미 실행 중이므로, Ollama를 실행하면:

  1. 메모리 부족으로 시스템이 느려지거나 불안정해질 수 있음
  2. 기존 서비스들의 성능에 영향을 줄 수 있음
  3. OOM(Out of Memory) 에러로 서비스가 강제 종료될 수 있음

Ollama 사용 조건

로컬 PC에서 사용하기

Ollama는 다음 환경에서 사용하는 것이 적합합니다:

  • 개인 PC/Mac: 8GB 이상 RAM 권장
  • 고성능 클라우드 서버: 8GB 이상 RAM, 전용 리소스
  • GPU 서버: NVIDIA GPU가 있으면 더욱 빠른 응답

설치 방법 (macOS):

# Homebrew로 설치
brew install ollama

# 서버 시작
brew services start ollama

# 모델 다운로드
ollama pull gemma3:4b

대안: 클라우드 AI 사용

서버 환경에서는 Claude, GPT, Gemini 같은 클라우드 AI를 사용하는 것이 효율적입니다:

  • 서버 리소스를 거의 사용하지 않음 (API 호출만)
  • 항상 최신 모델 사용 가능
  • 안정적인 응답 품질

문제 해결

AI가 응답하지 않음
  1. Settings에서 API 키가 올바르게 입력되었는지 확인
  2. 상단의 상태 표시등 확인 (녹색 = 온라인)
  3. 인터넷 연결 상태 확인
응답이 느림
  • Compare/Smart 모드는 여러 AI를 호출하므로 시간이 더 걸림
  • Ollama는 로컬 하드웨어 성능에 따라 다름
  • 네트워크 상태에 따라 응답 시간이 달라질 수 있음
